# 🌐 Internationalization (i18n) System

## 📋 Overview

The-Gargantuas-Video-Editor includes a complete internationalization system to support multiple languages.

## 🗂️ Structure

```
locales/
├── en.json         # English translations
├── it.json         # Italian translations (Italiano)
└── [future].json   # Add more languages here

utils/
└── i18n.py         # I18n Manager

config/
└── user_settings.json  # Saved user preferences (language, etc.)
```

## 🚀 How It Works

### 1. Translation Files (`locales/*.json`)

Each language has a JSON file with all translations organized hierarchically:

```json
{
  "app": {
    "title": "The-Gargantuas-Video-Editor",
    "subtitle": "AI-Powered Video Editing Suite"
  },
  "tabs": {
    "upscaler": "🎨 Upscaler",
    "lipsync": "👄 Lip Sync"
  }
}
```

### 2. I18n Manager (`utils/i18n.py`)

Manages loading and switching languages:

```python
from utils.i18n import get_i18n, t

# Get manager instance
i18n = get_i18n('it')  # Load Italian

# Translate keys
title = i18n.t('app.title')

# Or use shortcut
title = t('app.title')

# With variables
msg = t('lipsync.success', model='Wav2Lip')
# Result: "Lip sync completato con successo usando Wav2Lip!"
```

### 3. Settings Tab

Users can change language via **⚙️ Settings** tab:
1. Select language from dropdown
2. Click "Save Settings"
3. Refresh page to see changes

Preference is saved in `config/user_settings.json`.

## ➕ Adding a New Language

### Step 1: Create Translation File

Create `locales/[lang_code].json` (e.g., `es.json` for Spanish):

```json
{
  "app": {
    "title": "The-Gargantuas-Video-Editor",
    "subtitle": "Suite de Edición de Video con IA"
  },
  ...
}
```

**Tip**: Copy `en.json` and translate all values.

### Step 2: Register Language

Edit `utils/i18n.py`:

```python
AVAILABLE_LANGUAGES = {
    'en': 'English',
    'it': 'Italiano',
    'es': 'Español',  # Add here
}
```

### Step 3: Test

1. Run app
2. Go to Settings tab
3. Your new language should appear in dropdown

## 🔤 Translation Keys Reference

### App Level
- `app.title` - Application name
- `app.subtitle` - Tagline
- `app.init_message` - Initialization message
- `app.free_message` - Free app message

### Tabs
- `tabs.upscaler` - Upscaler tab name
- `tabs.lipsync` - LipSync tab name
- `tabs.settings` - Settings tab name
- `tabs.support` - Support tab name

### Upscaler
- `upscaler.title` - Tab title
- `upscaler.description` - Description
- `upscaler.upload_image` - Upload button
- `upscaler.model` - Model selector
- `upscaler.upscale_button` - Process button
- `upscaler.success` - Success message

### LipSync
- `lipsync.title` - Tab title
- `lipsync.description` - Description
- `lipsync.input_file` - Input file label
- `lipsync.audio_file` - Audio file label
- `lipsync.process_button` - Process button
- `lipsync.success` - Success message (supports `{model}` variable)

### Settings
- `settings.title` - Tab title
- `settings.language` - Language selector
- `settings.save` - Save button
- `settings.saved` - Success message

### Common
- `common.upload` - Upload
- `common.download` - Download
- `common.error` - Error
- `common.success` - Success
- `common.loading` - Loading...

## 🎨 Using Translations in Code

### In Main App
```python
from utils.i18n import get_i18n

i18n = get_i18n()
title = i18n.t('app.title')
```

### In Tabs
```python
from utils.i18n import t

label = t('upscaler.upload_image')
message = t('lipsync.success', model='Wav2Lip GAN')
```

### With Dynamic Content
```python
# Translation with variable
text = t('lipsync.init_model', model='SadTalker')
# English: "Initializing SadTalker..."
# Italian: "Inizializzazione SadTalker..."
```

## 🔧 Technical Details

### Language Detection Order
1. User's saved preference (`config/user_settings.json`)
2. Default: English (`en`)

### Caching
- Translation files loaded once at startup
- Changing language requires page refresh for full effect
- Settings are persisted across sessions

### Error Handling
- Missing keys return the key itself (e.g., `"app.missing"`)
- Missing language files fallback to English
- Invalid language codes default to English

## 📝 Best Practices

1. **Always use keys, never hardcode text**
   ```python
   # ❌ Bad
   gr.Button("Upload Image")
   
   # ✅ Good
   gr.Button(t('upscaler.upload_image'))
   ```

2. **Group related translations**
   ```json
   "upscaler": {
     "title": "...",
     "upload_image": "...",
     "upload_video": "..."
   }
   ```

3. **Use dot notation consistently**
   ```python
   t('tabs.upscaler')      # Tab name
   t('upscaler.title')     # Feature title
   t('upscaler.upload')    # Action
   ```

4. **Keep translations synchronized**
   - When adding a key to `en.json`, add it to ALL language files
   - Use English text as placeholder if translation not ready

5. **Use variables for dynamic content**
   ```json
   {
     "message": "Processing {filename} with {model} model"
   }
   ```
